# ERC-7710 delegated-permissions interop

AgentWorth's `Mandate` is the off-chain authority object: scoped, capped, expiring,
revocable spend authority granted by an operator. ERC-7710 (the MetaMask
delegation-framework shape) is the *on-chain* enforcement of permission bounds via
**caveat enforcers**. The `@general-liquidity/agentworth/erc7710` module is the bridge:
it expresses a mandate as a `Delegation` + `Caveat[]`, computes the EIP-712 delegation
hash, and signs / verifies it (ECDSA secp256k1).

A mandate and a delegation are complementary. The mandate is what the AgentWorth gate
enforces in software, deterministically and replayably. The delegation is what an on-chain
DelegationManager + enforcer contracts enforce when the same authority is exercised
on-chain. This module lets you derive the second from the first without re-stating the caps.

## Mandate ↔ Delegation mapping

| Mandate field | Caveat enforcer | Terms encoding |
|---|---|---|
| `expiresAt` | `TimestampEnforcer` | 32 bytes: `uint128 afterThreshold ‖ uint128 beforeThreshold` (non-inclusive). `expiresAt` → `beforeThreshold`. |
| `perTxCap` (no token) | `NativeTokenTransferAmountEnforcer` | `abi.encode(uint256 allowance)` (32 bytes). |
| `perTxCap` (ERC-20 `token`) | `ERC20TransferAmountEnforcer` | `address token` (32-padded) ‖ `uint256 allowance` (64 bytes). |
| `perPeriodCap` + `period` + `grantedAt` (ERC-20) | `ERC20PeriodTransferEnforcer` | 116 bytes packed: `address token(0:20) ‖ uint256 periodAmount(20:52) ‖ uint256 periodDuration(52:84) ‖ uint256 startDate(84:116)`. |
| `perPeriodCap` + `period` + `grantedAt` (native) | `NativeTokenPeriodTransferEnforcer` | 96 bytes packed: `uint256 periodAmount ‖ uint256 periodDuration ‖ uint256 startDate`. |
| `scope.kind === "allowlist"` | `AllowedTargetsEnforcer` | Packed 20-byte addresses (`len % 20 == 0 && len != 0`). |
| `allowedRails` | — | No on-chain enforcer; rails are an off-chain concept. |

`period` → seconds via `PERIOD_SECONDS` (`day=86400`, `week=604800`, `month=2592000`),
overridable per call. `authority` is set to `ROOT_AUTHORITY` (a root grant). `salt` defaults
to a deterministic FNV-1a hash of the mandate id (override via `opts.salt`). The returned
delegation is unsigned (`signature: "0x"`) until you call `signDelegation`.

> **Allowlist values must be 0x addresses.** `AllowedTargetsEnforcer` constrains the *call
> target* (the contract/recipient address). A mandate's `scope.allowlist.values` are opaque
> payee ids in AgentWorth; to map them on-chain they must be Ethereum addresses.

## Enforcer addresses are deployment-specific

Enforcer contracts are deployed per chain. The module never hardcodes an address — you pass
an `EnforcerAddresses` record for your target deployment:

```ts
import { mandateToDelegation, signDelegation, verifyDelegation } from "@general-liquidity/agentworth/erc7710";

const enforcers = {
  timestamp: "0x…",
  erc20TransferAmount: "0x…",
  erc20PeriodTransfer: "0x…",
  allowedTargets: "0x…",
};

const delegation = mandateToDelegation(mandate, {
  delegate: agentEoa,      // who may act
  delegator: operatorEoa,  // who grants (and signs)
  enforcers,
  token: usdcAddress,      // omit for native-token caps
});

const domain = { chainId: 8453, verifyingContract: delegationManagerAddress };
const signed = await signDelegation(delegation, domain, operatorPrivKey);
const { ok, signer } = await verifyDelegation(signed, domain);
```

## EIP-712 hashing

```
CAVEAT_TYPEHASH      = keccak256("Caveat(address enforcer,bytes terms)")
DELEGATION_TYPEHASH  = keccak256("Delegation(address delegate,address delegator,bytes32 authority,Caveat[] caveats,uint256 salt)Caveat(address enforcer,bytes terms)")

caveatHash     = keccak256(abiEncode(CAVEAT_TYPEHASH, enforcer, keccak256(terms)))
caveatsHash    = keccak256(encodePacked(caveatHash…))
structHash     = keccak256(abiEncode(DELEGATION_TYPEHASH, delegate, delegator, authority, caveatsHash, salt))
domainSeparator = keccak256(abiEncode(EIP712DOMAIN_TYPEHASH, keccak256(name), keccak256(version), chainId, verifyingContract))
typedDataHash  = keccak256(0x1901 ‖ domainSeparator ‖ structHash)
```

All signed fields are fixed 32-byte words, so the ABI encoding is a manual left-pad — no ABI
library is required. The domain defaults to `name="DelegationManager"`, `version="1"`. Note
that `Caveat.args` is **excluded** from the signed hash (it is execution-time data).

The hashing is cross-checked against viem's `hashTypedData` in the test suite (golden vector).

## Verification scope: EOA only

`verifyDelegation` recovers the signer from the 65-byte `r ‖ s ‖ v` signature (`v = 27/28`)
and checks it equals `delegator`. This covers EOA (externally-owned-account) delegators.

**EIP-1271 (smart-contract delegators) is out of scope** — contract signature verification
requires an on-chain `eth_call` to the delegator's `isValidSignature`, which this pure module
cannot perform. Use an RPC client for that path.

## Optional crypto dependency

The pure mapping/encoding functions (`mandateToDelegation`, the `encode*Terms` helpers,
`abiWord`, hex/bytes utilities) use **no crypto** and load no dependency. Only the hash / sign
/ verify functions dynamically import `@noble/hashes` and `@noble/curves`, which live in
`optionalDependencies`. If they are absent, the crypto functions throw a clear install error;
the AgentWorth core gate never pulls a crypto dependency.

## Relationship to ADP (Agent Disclosure Protocol)

ERC-7710 *enforces* permission bounds on-chain; ADP only *discloses* them. The two are
orthogonal: an ADP disclosure can reference a delegation hash via its generic fields, so no
dedicated ADP module is needed for this interop.
